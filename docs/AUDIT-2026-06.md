# Full-Codebase Audit — June 2026

Five-stream independent review of the project's research, documentation, engine
code, web/session security, and test suite. Every artifact in this repo —
research, design notes, the build spec, the training manual, the YAML vignette
library, and the code — was written by AI models. This audit treats nothing as
sacred and validates against external references where possible.

## Stream C — Engine code

### C1 — CRITICAL — `perturbations.py:101-104` J3 zonal polynomial is wrong

Coded as `(5 sin³φ − 3 sin φ)` with trailing `0.4`; Vallado Eq. 8-30 gives
`(7 sin³φ − 3 sin φ)` with trailing `0.6`. The `(5 sin³φ − 3 sin φ)` form is
the Legendre P₃ inner polynomial — correct for the **potential** but not for
its **gradient** (the force), which is what we need. Effect on current
simulation: **none** — `perturbations.py` is preview-only today, used by no
active propagator. But the module is documented as "composed by future
high-fidelity propagators," so fix now before that wires up.

Add a regression test that pins J3 acceleration against a reference state
computed from Vallado / Curtis.

### C2 — MAJOR — `orbit.py:57-59` `classify_regime` GEO bounds wrong

```python
if 33_000e3 <= a_m - 0 <= 37_000e3 or 34_000e3 <= apogee <= 37_000e3:
    return "GEO"
```

GEO is `a ≈ 42,164 km`, not 33–37 Mm. Real GEO satellites currently classify as
**MEO**. UI-labelling impact only (not physics). Fix:

```python
if 41_000e3 <= a_m <= 43_000e3 or 35_000e3 <= apogee <= 37_000e3:
    return "GEO"
```

### C3 — MAJOR — `orders.py` booking dicts leak after execute

`_pass_bookings`, `_order_pass`, and `_order_sensor` are populated on plan but
never popped on execute (only on cancel). Slow memory leak proportional to
executed-command count. Fix: at execute-time in `_h_observe` / `_h_command` /
`_h_maneuver`, pop the order id from the three booking dicts. Also rebuild
`_pass_bookings` in `session/manager.py::_rebind` for symmetry with
`_sensor_bookings`.

### C4 — MINOR cluster

- `orbit.py:62` `period_s` doesn't guard `a_m > 0` (sqrt of negative → NaN).
- `maneuver.py:230` plane-change docstring slightly misleading (Rodrigues is
  mathematically exact but physically gives a true plane change only at the
  node — magnitude formula `2v sin(Δi/2)` is correct everywhere; clarify).
- `perturbations.py:54-65` `atmospheric_density` has ~12% discontinuities at
  bracket boundaries (uses bracket-start scale height); fidelity wart, not a
  bug.
- `perturbations.py:131-147` SRP `reflectivity=1.3` default is `Cr`, not albedo

### C5 — Determinism: PASS

use outside `engine/rng.py`, no `set()` construction in event-producing paths.
All `acos`/`asin` calls correctly clamped to `[-1, 1]`. GMST polynomial matches
IAU-82 (Vallado Eq. 3-45). Hohmann Δv signs match Curtis §6.3. Drag formula and
SRP direction correct. J4 axisymmetric coefficient matches Vallado Eq. 8-31.

## Stream D — Web / session security

LAN-deployed trust model is documented, but several findings go **beyond** the
documented trust boundary or are independently exploitable.

### D1 — HIGH — Stored XSS via inject message text

`app.js:946` `messages.map((m) => \`<li>${m.text}</li>\`)` interpolates inject
`m.text` with no escaping. Any LAN participant can `POST
/api/sessions/{sid}/inject` (server.py:220 — no cell check) with HTML/script
payload; script executes in every other operator's browser.

**Fix:** `<li>${esc(m.text)}</li>`. `esc()` already exists in `app.js`.

### D2 — HIGH — Stored XSS via effect target / TLE asset ids

`app.js:945, 938, 940` interpolate `e.target`, `e.symptom`, and `a.id`
unescaped. `a.id` is **attacker-controlled** via `POST /force/tle`
(`TleRequest.id` stored verbatim, `manager.py:250`).

**Fix:** `esc()` all of these, AND validate `TleRequest.id`,
`OrderRequest.actor`, `OrderRequest.target` server-side against
`^[A-Za-z0-9_.-]{1,64}$`.

### D3 — MEDIUM — Vignette title / inject-label XSS

`app.js:157, 181, 199, 1299` interpolate vignette title and inject labels
unescaped. Lower risk (server-side YAML), but a hostile vignette file injects.
Fix: `esc()`.

### D4 — HIGH — Path traversal in `load_vignette`

`spacesim/content/vignette.py:103-117` does `path = Path(path_or_id); if
path.exists(): yaml.safe_load(path.read_text())`. The argument flows from
unsanitized client input via `GET /api/vignettes/{vid}/tutorial`, `POST
/api/sessions {vignette_id}`, and `POST /api/sessions/load_save
state["vignette_id"]`. Even though most files won't parse into a `Vignette`,
this is an arbitrary file-open primitive (existence probing, error-based
disclosure).

**Fix:** reject `path_or_id` containing `/`, `\`, or `..`; resolve only as
`VIGNETTE_DIR / f"{id}.yaml"`; assert `resolved.parent == VIGNETTE_DIR`.

### D5 — HIGH — Ground-truth endpoints with no cell binding

Cell identity is client-side trust (documented). But these endpoints expose
**ground truth with no cell argument at all**, so any LAN client reads them
directly:

- `/godview` → full `world.model_dump()`
- `/eventlog` → full event history (replayable ground truth)
- `/objectives` → both cells' objectives
- `/save` → full state
- `/aar`, `/aar/at`, `/aar/objectives`, `/aar/export.*` → full replay

Cell-scoped endpoints (`/scene/{cell}`, `/telemetry/{cell}/{asset}`,
`/view/{cell}`, `/orders/{cell}`, `/activity/{cell}`, `/conjunctions/{cell}`)
trust the `{cell}` path segment, and `_owns()` treats `cell == "white"` as
see-all. So a Red browser fetching `/scene/blue` or `/scene/white` bypasses
fog entirely.

**This is foundational.** The documented trust model says cell identity is
client-side, but the no-cell ground-truth endpoints arguably exceed even that
weak model. **Two options:**

- **(a) Minimal — gate the godview/eventlog/save/aar/objectives endpoints
  behind a per-session White token issued at session-create time.** Cell-scoped
  endpoints continue to trust the path segment. Preserves the LAN-trust model
  for Red/Blue but stops Red/Blue from trivially reading ground truth via the
  no-cell endpoints. ~50 lines of code + 3 new tests.
- **(b) Full — every cell-scoped endpoint requires a cell-bound token issued
  at join.** Substantial work (~200 lines) and changes the LAN UX. Deferrable.

Recommended: **(a) now**, **(b) tracked in FUTURE-WORK**.

### D6 — MEDIUM — Unbounded telemetry series `n`

`server.py:411-414` passes `n` straight into `telemetry.series` which loops
`range(n)`. `GET /telemetry/{cell}/{asset}/{param}?n=1000000000` hangs a
worker.

**Fix:** `n: int = Query(120, ge=2, le=2000)`.

### D7 — MEDIUM — Unbounded sessions and `load_save` replay

`POST /api/sessions` and `/load_save` create new managers with no cap or
eviction; `_sessions` grows forever. `load_save` accepts an arbitrary `state`
dict whose `eventlog`/`pending`/`orders` are replayed with no size limit — a
large event log is an amplified compute/memory bomb, unauthenticated.

**Fix:** cap live sessions (e.g., 32, LRU-evict); cap `len(eventlog.entries)`
in `from_state` (e.g., 100_000).

### D8 — LOW–MEDIUM — `advance_to` / `step` / `undo` unbounded ints

Compute is largely bounded by `horizon`, but clamp `t` to `[now, horizon]`,
`dt_sim_s` to `[1, 86400]`, `n` to `[1, 100]` to be safe.

### D9 — LOW — No CORS / no request-size limit / binds `0.0.0.0`

Inherent to the LAN-trust deployment, but document as "trusted LAN only" and
ideally add a `--lan-secret` shared key. Add a body-size limit (e.g., 4 MB) to
blunt D7's `load_save`.

### D10 — LOW — Concurrency window between catch_up and read

Several read paths call `self.catch_up(session)` (which acquires the lock) and
then read `mgr.xxx()` **outside** the lock (e.g., `list_orders`, `get_view`,
`get_scene`, `cell_activity`, `conjunctions` in server.py:317-326 iterates
`mgr.world.conjunctions` lock-free). A concurrent mutation can cause `dict
changed size during iteration`. **Fix:** hold `mgr._lock` for the whole read
(catch_up + snapshot).

## Stream E — Test suite / TDD

### E1 — CRITICAL — Determinism gate covers stubs, not domain handlers

`spacesim/tests/test_determinism.py` is a genuine Hypothesis property test
(200 examples, byte-identical replay + snapshot + save round-trip). But the
event vocabulary is only the three Phase-1 generic handlers (`inc_counter`,
`random_roll`, `set_value`). It exercises **none** of `execute_effect`,
`execute_maneuver`, `execute_observe`, `execute_downlink`, `execute_command`,
`ssn_collect`, `ssn_deliver`, `recovery_confirm`, `recovery_finish`, `bus_tick`,
`telemetry_contact`, `isr_downlink`.

The "permanent gate" therefore does not gate the systems most likely to
introduce nondeterminism (RNG draws in effect resolution, dict-iteration order
in SSN sensor selection, float accumulation in bus ticks).

**Fix:** add `test_determinism_domain_events` — Hypothesis property test that
randomly composes real orders + SSN requests + recovery on a small fixed force,
advances, and asserts `sim.replay()` is byte-identical. Keep the existing test
as the fast-feedback gate.

### E2 — CRITICAL — No systematic fog-of-war proof

Fog is checked piecemeal on 6 endpoints (`view`, `telemetry`, `scene` once,
`activity`, `ssn`, `orders`). No assertion that Red can **never** see Blue
ground truth across ALL read endpoints. `get_series`, `scene/red`,
`windows_ahead`, `conjunctions`, `brief`, `recovery_status`, `coaching` have
no fog assertion. A new endpoint that forgets `_owns()` ships green.

**Fix:** `test_fog_all_endpoints_red_cannot_see_blue` — parametrized over every
read endpoint; load Vignette 1; for each endpoint assert no Blue asset id
appears in the Red response (and vice versa).

### E3 — MAJOR — Import-guard bypasses

`spacesim/tests/test_import_guard.py` AST-scans correctly for `import random`,
`datetime.now`, etc. **Bypasses NOT caught:**

1. `from time import monotonic; monotonic()` — the wall-clock detector only
   matches `time.monotonic` attribute chains. `from time import sleep` also
   undetectable.
2. `importlib.import_module("requests")` / `__import__("httpx")` — dynamic
   imports of forbidden roots are invisible.
3. `from numpy.random import default_rng` — `"numpy.random" != "random"` and
   doesn't `startswith("random.")`, so NumPy's global RNG could leak in.
4. Relative-import escape: `_imported_roots` skips `ImportFrom` when
   `node.level != 0`.

**Fix:** extend the guard to track `from time import X` / `from random import
X` names + scan `Call` on bare names; flag `importlib.import_module` and
`__import__`; flag any root containing `.random`; flag `level != 0` references
to forbidden roots. Add a fixture-based test that proves each bypass is
caught.

### E4 — MAJOR — Skip-masked Skyfield oracle

`test_reference_skyfield.py` is the only test that validates orbital propagation
against an external reference (Skyfield). It is `pytest.skip`'d when Skyfield
isn't installed; CI can pass with 0 reference checks. Either make Skyfield a
hard test-time dep (it's already documented) or have the skip be `pytest.fail`
in CI mode.

### E5 — MAJOR — Tautological / snapshot-style tests

- `test_inject_library.py:58` `assert a is not b or len(a) == len(b)` — pure
  tautology, can't fail. Replace with the real caching contract (`a is b`).
- `test_orbit.py:42` `assert ||r_period - r0|| < 60_000.0` — 60 km tolerance
  pins the code's own current J2 drift; not derived from theory.
- `test_jam.py`, `test_isr.py`, `test_sigint.py`, `test_cyber.py` value
  assertions are change-detectors against the module's own constants, not
  correctness checks. Acceptable but should not be counted as physics coverage.
- `test_telemetry.py` thresholds are copied-from-output.
- `test_isr.py:315,321` accept-either assertions (`status in ("queued",
  "executed")`, `>= 0  # may be 0`) weaken the check.

### E6 — MAJOR — Untested third-body perturbation

`perturbations.third_body_acceleration` (line 120) has no test. Drag/J3/J4/SRP
are covered; third-body Sun/Moon is not.

### E7 — MINOR cluster — coverage gaps

- `geometry.py` — no edge cases at lat ±90 (poles), lon ±180 (antimeridian),
  GMST wrap.
- `conjunction.py` — only an indirect smoke test; no no-encounter or screening-
  volume boundary case.
- `redai.py` — three presets' divergence is asserted but not malformed-doctrine
  handling.
- `test_action_log_file_roundtrip` — M7 "written at exercise end and proven
  sufficient to reconstruct offline" is asserted in memory, not on disk.

## Foundational decisions — locked in

**F1 — fog-of-war / ground-truth endpoints (D5):** **Document, don't change.**
The LAN-trust model assumes cooperative participants; ground truth is
voluntarily fog-respected. Add an explicit note to CLAUDE.md, README, and the
build spec. XSS (D1–D3), path traversal (D4), DoS bounds (D6–D8), and
concurrency (D10) are still fixed in code — those are independent of trust.

**F2 — 48-satellite cap (B1):** **Soften the spec; add a clock-lag watchdog.**
The 24/48-sat numbers were only sized for typical White-Cell facilitator
hardware, not as a fundamental engine limit. Reword as soft guidance for
Claude-generated sample vignettes; user-authored vignettes may tailor to their
hardware. **NEW FEATURE:** add a server-side clock-lag watchdog that compares
the running sim time against wall-clock anchor drift; when drift exceeds a
threshold for sustained seconds, surface a White-Cell warning (`hardware
insufficient for this scenario`). Implements the cap as an empirical-not-static
guard.

**F3 — Bylina mis-typing (A3):** **Rename `RED-JAM-BYLINA` → `RED-JAM-TIRADA`.**
Tirada-2 is the unambiguous Russian SATCOM jammer at the Dzhankoy site; the
rename removes the typology error. Also correct the research-doc claim that
Bylina is a GNSS jammer.

**F4 + F5 — research completeness (mix of both):**
- Add a paragraph on Russian orbital nuclear-ASAT / EMP coverage to
  `02-doctrine-non-western.md` + a taxonomy note in `03-counterspace-taxonomy.md`
